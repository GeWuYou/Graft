# AI Governance

This directory is for repository-wide AI workflow and AI-tooling governance.

## Use This Directory For

- AI task recovery and topic-governance rules
- MCP and AI-tooling adoption guardrails
- AI code-generation and AI review governance
- repository-wide documentation rules that primarily exist to support AI-assisted development

## Boundaries

- These documents remain repository-wide design truth, not topic-local notes.
- Do not use this directory for one-off recovery artifacts; those stay under `ai-plan/public/<topic>/`.
- Do not use this directory for runtime implementation guidance that belongs under backend, frontend, or domain
  directories.
