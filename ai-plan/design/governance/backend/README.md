# Backend Governance

This directory is for server-authority and backend-led engineering guardrails.

## Use This Directory For

- API boundary and compatibility governance
- backend security and trust-boundary rules
- database, query, and test-maintainability governance
- backend-side contract and magic-value governance

## Boundaries

- Use `../../domains/` when a design note mainly belongs to one capability area.
- Use `../platform/` when the rule is cross-cutting platform governance rather than backend-authority engineering
  discipline.
