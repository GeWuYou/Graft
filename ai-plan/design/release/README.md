# Release

This directory stores release-policy design for build identity, configuration, migration, upgrade, support, and
versioning boundaries.

## Use This Directory For

- release-policy rules that apply across the repository
- versioning, support-boundary, and migration-policy design

## Boundaries

- Keep runtime implementation or delivery tracking out of this directory.
- Use `ai-plan/roadmap/` for staged execution planning and `ai-plan/public/<topic>/` for active recovery state.
