# Graft 前端视觉设计规范

## 1. 目标

这份规范把 Graft 的前端视觉基线固定下来，作为 `web` 的设计真值补充。

目标不是“做得更花”，而是让所有后台页面都满足：

- 结构统一
- 状态清晰
- 主题可切换
- 组件可复用
- AI 可稳定生成

## 2. 视觉定位

Graft 是组合式后台平台控制台，因此视觉策略应偏向：

- 控制台感
- 信息密度高但不拥挤
- 品牌感来自 token、节奏和层次，不来自大面积装饰
- 允许少量科技感，但不能偏营销页

保留项：

- 当前主题工作台的悬浮按钮栏和 token 编辑感可以作为正向样本

禁止项：

- 过度渐变
- 复杂背景纹理
- 通用 SaaS 营销风 Hero
- 与 TDesign 背离的第二套视觉系统

## 3. 页面骨架

### 3.1 壳层页

统一使用：

- 顶部导航
- 侧边导航
- 面包屑
- 内容区
- 用户操作区

### 3.2 标准业务页

默认顺序：

1. 页面标题与摘要
2. 筛选区
3. 列表 / 卡片 / 图表主内容
4. 详情抽屉或弹窗
5. 空态、异常态与提示区

### 3.3 仪表页

推荐结构：

- 顶部概览卡
- 趋势或指标图卡
- 依赖状态卡
- 运行时明细卡

### 3.4 登录页

推荐结构：

- 左侧品牌与说明
- 右侧表单
- 底部版权与辅助入口

## 4. 颜色与状态

- 页面色彩优先使用 `--td-*` token。
- 状态语义必须映射到 TDesign 语义色。
- 图表颜色必须随 mode、brand theme 和容器变化同步。
- 只有 token 缺失时，才允许十六进制兜底。

## 5. 组件组合

优先组合：

- `Card` + `Table` + `Form`
- `Drawer` + `Form`
- `Dialog` + `Form`
- `Tag` + `Alert`
- `Tabs` + `Card`
- `Result` + 空态插画

原则：

- 列表页优先用表格
- 编辑交互优先用抽屉或弹窗
- 详情页优先用卡片分区
- 批量操作必须显式可见

## 6. 模板落点

可复用参考模板放在：

- `ai-plan/design/graft-design-system/shell.md`
- `ai-plan/design/graft-design-system/auth.md`
- `ai-plan/design/graft-design-system/overview-dashboard.md`
- `ai-plan/design/graft-design-system/list-form-detail.md`

这些模板只作为参考资产，不是运行时依赖，也不是页面白名单。

## 7. 生成约束

AI 生成前端页面时，应先锁定：

- 页面类型
- 主信息层级
- 组件组合
- 状态表达
- 主题 token 依赖

再开始写代码。

## 8. 验收标准

一页是否符合本规范，至少看这几件事：

- 组件结构是否清晰
- 状态是否可读
- 颜色是否 token 驱动
- 是否能在浅色/深色模式下成立
- 是否仍然像 Graft，而不是 demo 模板
